## design.md A.2 增加 source_refs 设计
- 在 `docs/design.md` 的 A.2「切分成检索单元」步骤3 给检索单元 JSON 新增字段 `source_refs`：本单元讲到/引用的课文原文原句 + 段落号，形如 `[{"para":6,"quote":"..."}]`。
- 定位：附加锚点，不替代 `text`（讲解文本仍是主体）。两大作用：① 检索锚点（按原文/段落号精确召回，配合 B.2 的 where 过滤）；② 防幻觉素材（生成时喂权威原文，避免课文背错）。原文=事实锚，老师讲解=风格锚。
- 前提：DeepSeek 训练数据中已包含常见语文课文原文，可直接凭记忆标注，不依赖外部课文原文库。不常见课文可能不准或留空，靠人工复核兜底。
- 对齐粒度：段落级 + 引用句即可，不逐句严格对齐；不确定的留空，靠人工复核环节补。
- 抽取方式：在补 summary/entities 的同一次模型调用里顺带产出。

## design.md 新增第 11 章：音视频/PPT 高亮方案
- 用户需求：讲解时原文句子高亮、重点词高亮。
- 两条路线：A）PPT/网页（轻量，先做）；B）视频 MP4（重量，后做）。
- 核心：B.2 输出结构化 JSON（sentences + keywords + narration 时间轴），渲染层按时间轴高亮。
- 时间轴来源：① 伪时间轴（段落序号，纯文本）；② TTS 后语音对齐（精确秒数）。
- 推荐顺序：结构化 JSON → HTML 网页 → python-pptx PPT → TTS+对齐 → 视频。
- 已追加到 docs/design.md 末尾。

## design.md 第 11 章新增 11.4.2.1：关键词高亮时间推算
- 问题：DeepSeek 生成讲解时不知道音频信息，无法输出时间标记。
- 方案：位置比例推算——关键词在讲解文本中的位置 / 总长度 × 音频时长 = 高亮时间。
- 可提前高亮：`start = time - 0.5`。
- 数据结构新增 `keyword_timings` 字段：`[{word, note, time, start}, ...]`。
- 更新 11.4 数据结构、11.5 视频流程、11.6 字段说明，都加上关键词高亮时间引用。
