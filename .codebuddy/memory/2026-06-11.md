# 2026-06-11

## script/transcribe_dir.py bug 修复
检查并修复 3 个 bug：
1. `process_one` 占位 `touch` 前未建父目录 → 子目录 mp3 抛 FileNotFoundError 崩溃。已加 `out_dir.mkdir(parents=True, exist_ok=True)`。
2. 中断遗留的 0 字节占位 txt 会被 `process_one` 的 `touch(exist_ok=False)` 当成 FileExistsError → 永久 skip。已改为：非空才 skip，0 字节复用重处理。
3. `build_tasks` 按 `txt.stem` 去重，跨子目录同名文件互相误判跳过。已改为按实际目标路径 `out_txt` 判断完成。

lint 剩余的都是 basedpyright 严格模式类型噪音，非真实 bug。

## 占位机制修正（接上）
之前第 2 条"0 字节复用重处理"破坏了多进程占位（A 刚占位未写完，B 见 0 字节也去处理 → 重复）。已分两阶段解决：
- 启动阶段 `build_tasks`（单进程）：0 字节遗留占位直接 unlink 重入队，安全。
- 运行阶段 `process_one`（多进程）：目标已存在即 skip，否则 `touch(exist_ok=False)` 原子抢占。
约束：仅支持「单脚本 --workers N」；手动并发多个脚本实例时启动清理会误删他人占位，需改 mtime 陈旧判断/锁文件。
